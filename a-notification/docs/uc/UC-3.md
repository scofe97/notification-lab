# UC-3. 알림 이력 조회 — 리뷰·연구 노트

NCP 발송 이력을 중계 조회하는 흐름입니다. UC 명세는 [../02-actors-usecases.md](../02-actors-usecases.md) §UC-3에 있습니다.

- **상태**: 미구현 (`notification.history` 패키지에서 다룸. UC-5 아카이빙이 색인한 이력을 조회)
- **관련 FR**: FR-10
- **주 액터**: 사용자

---

## 1. 개념 역링크 (원본 분석 문서)

원본 루트: Drive `비공개 분석 노트/`

| 기술 | 원본 분석 문서 | UC-3에서의 역할 |
|------|----------------|-----------------|

---

## 2. 코드 리뷰 렌즈 (구현 후 채움)

핵심은 **채널마다 쿼리 형식이 달라 매퍼를 선택**한다는 점입니다. 전략(Strategy)/팩토리 패턴이 어떻게 채널 분기를 캡슐화하는지 봅니다.

- 볼 것 (구현 시): `QueryParamMapperFactory`가 채널로 매퍼를 고르는 방식, OpenSearch 쿼리 조립, 이력 응답 형상.

---

## 3. 직접 관찰·실측법 (구현 후 채움)

- UC-5가 색인한 이력을 `GET .../history?channel=SMS`로 조회해 채널별로 다른 쿼리가 나가는지 OpenSearch 로그로 확인.
