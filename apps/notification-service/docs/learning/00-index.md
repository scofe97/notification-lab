# 구현 후 이해 기록

> 구현을 끝낸 뒤 코드와 검증 기록을 대조해, 다음 작업으로 넘어가기 전에 흐름을 자기 언어로 설명할 수 있게 만드는 기록입니다. 각 문서는 예측·읽기·추적·실측·인출의 다섯 단계를 시간 순서로 남깁니다.

이 폴더는 구현을 끝낸 뒤 코드·문서·검증 기록을 대조하고, 다음 작업으로 넘어가기 전에 흐름을 설명할 수 있게 만드는 학습 기록입니다. 기능 명세는 [유스케이스](../02-actors-usecases.md), 구현 검토 절차는 [UC 리뷰 노트](../uc/00-index.md), 진행 상태는 [PROGRESS.md](../../PROGRESS.md)에 둡니다.

각 문서는 다음 순서로 진행합니다.

1. 코드 보기 전 목적과 실패 가능성을 예측합니다.
2. 정상 경로를 실제 파일·심볼 순서로 읽습니다.
3. 실패와 경계 조건을 코드와 설정으로 추적합니다.
4. 다섯 질문과 자기 설명으로 이해를 확인합니다.

문서의 사실은 실행으로 확인한 것, 코드에서 추론한 것, 아직 검증하지 않은 것으로 나눕니다. 불일치를 찾으면 이 폴더에 후속 검증으로 남기며, 학습 과정에서 코드를 자동으로 바꾸지 않습니다.
