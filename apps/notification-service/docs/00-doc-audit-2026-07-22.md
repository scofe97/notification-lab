# 문서·UC 구성 검증 리포트 (2026-07-22)

프로젝트의 (1) UC 독립성·학습 순서, (2) 문서 구분, (3) 추가 필요 항목을 점검하고 발견한 불일치를 코드 기준으로 정합화한 기록입니다. 정합화 방침은 "실제 구현은 코드가 1차 자료"입니다.

## 1. 결론 요약

- **UC 독립성**: 양호. UC-1~5는 트리거 축(Kafka·REST·사용자·스케줄러)으로 독립적이고, 의존 관계(UC-4→UC-1 입력, UC-2→UC-1 재사용, UC-3→UC-5 색인)가 실재한다. 각 UC 문서 상단에 "학습 순서" 줄로 선행·후속을 명시했다.
- **학습 순서**: 번호와 진행 순서가 섞여 혼란 소지가 있었다. → **번호 = 명세 ID(불변), 진행 순서 = ROADMAP이 SSOT**로 못박고 각 UC에 의존 링크를 달아 해소.
- **문서 구분**: 4계층(설계 `docs/` → 리뷰 `uc/` → 이해 `learning/` → 개념 `concepts/`)이 역할별로 잘 분리돼 있다. 단 일부 문서가 리팩토링·구현을 못 따라잡은 stale이 있어 수정했다.
