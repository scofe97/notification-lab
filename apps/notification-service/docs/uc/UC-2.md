# UC-2. 외부 시스템 REST 발송 — 리뷰·연구 노트

외부 시스템이 REST로 직접 발송을 요청하는 흐름입니다. UC 명세(주 흐름·대안 흐름)는 [../02-actors-usecases.md](../02-actors-usecases.md) §UC-2에 있습니다.

- **상태**: 미구현 (UC-1 발송 파이프라인 완성 후 후속. `notification.send` 패키지에 REST 진입점 추가 예정)
- **관련 FR**: FR-6
- **주 액터**: 외부 시스템

---

## 1. 설계 초점

| 기술 | UC-2에서의 역할 |
|------|-----------------|
| OpenFeign | 수신자 조회 API 호출 |
| REST 계약 | 외부 요청을 내부 발송 명령으로 변환 |

---

## 2. 코드 리뷰 렌즈 (구현 후 채움)

핵심은 UC-1과의 차이입니다: UC-1은 **Kafka가 트리거**, UC-2는 **REST 요청이 트리거**이고 중간에 **수신자 조회(OpenFeign)**가 하나 더 낀다는 점.

- 볼 것 (구현 시): 기본·비즈니스 검증 위치, 채널별 이벤트 변환, 조직 API 조회 결과를 발송 단위로 집계하는 로직, 전체 결과 → 응답 코드 결정 규칙.

---

## 3. 직접 관찰·실측법 (구현 후 채움)

- WireMock에 수신자 조회 API와 발송 API를 두고, REST 호출 한 번이 조회 → 발송으로 이어지는지 요청 카운트로 확인.
