# UC-5. 로그 아카이빙 — 리뷰·연구 노트

스케줄러가 주기적으로 이력을 파일로 내보내는 흐름입니다. UC 명세는 [../02-actors-usecases.md](../02-actors-usecases.md) §UC-5에 있습니다.

- **상태**: 구현 + 스모크 검증 완료 (2026-07-22). 색인(FR-8)은 발송 직후 실시간 기록(best-effort — 기록 실패가 발송 재시도·중복을 유발하지 않도록 흡수), export는 매일 03:00 cron + 수동 재실행 API(`POST /api/history/archive/{day}`)
- **관련 FR**: FR-8·FR-11
- **주 액터**: 스케줄러 (시간 트리거)
- **학습 순서**: 선행 → [UC-1](UC-1.md)(발송이 남긴 이력을 색인·아카이빙하므로 발송 흐름을 먼저 안다). 후속 → [UC-3](UC-3.md)(이 색인을 조회한다). 번호는 명세 ID이며 진행 순서의 SSOT는 [ROADMAP](../../../../ROADMAP.md)이다.

## 2. 코드 리뷰 렌즈 (구현 후 채움)

핵심은 **시간이 액터**라는 점입니다. 사람 요청이 아니라 `@Scheduled` cron이 트리거이고, 실패 시 재시도·중복 실행 방지(멱등성)를 어떻게 다루는지가 관전 포인트.

- 볼 것 (구현 시): cron 표현식, 전일 인덱스 선택 로직, export 파일 형식, 배치 실패 시 처리.

## 3. 직접 관찰·실측법 (구현 후 채움)

- cron을 짧은 주기로 임시 변경해 트리거를 관찰하고, export 파일이 생기는지·UC-3 조회 대상과 일치하는지 확인.
